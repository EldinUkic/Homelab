## Topologie

<p align="center">

</p>

Der Standort Deutschland ist der administrative Hauptknoten meiner Infrastruktur. Hier laufen beide Proxmox-Nodes, die zentrale Firewall sowie der lokale Access Point. Der ISP-Router (FritzBox) arbeitet nur als Modem — Routing, NAT und VPN übernimmt ausschließlich die OPNsense.

<hr style="border: none; height: 2px; background: linear-gradient(to right, transparent, #555, transparent); margin: 30px 0;">

## Logischer Datenfluss

```text
[ Internet / ISP ]
        |
   [ FritzBox ]            ← Modem / DSL-Einspeisung
        |
   [ OPNsense ]            ← Firewall · NAT · VPN · Gateway (10.10.10.1)
        |
   [ Switch 1 ]            ← Core-Verteiler
    /        \
[Switch 2]  [Access Point] ← Arbeitsplatz-Switch · WLAN (10.10.10.2)
  /    \
[Node 1] [Node 2]          ← Proxmox 24/7 (10.10.10.10) · Proxmox On-Demand (10.10.10.20)
```

<hr style="border: none; height: 2px; background: linear-gradient(to right, transparent, #555, transparent); margin: 30px 0;">

| Segment           | Netz            | Zweck                                      |
| ----------------- | --------------- | ------------------------------------------ |

## VPN-Verbindung zu Serbien

Deutschland ist über einen WireGuard Site-to-Site Tunnel mit dem Standort Serbien verbunden. Die OPNsense übernimmt hier die VPN-Terminierung auf beiden Seiten.

| Parameter             | Wert                                                    |
| --------------------- | ------------------------------------------------------- |
| **Tunnel-Netz**       | `10.0.0.0/24`                                           |
| **Deutschland-Seite** | `10.0.0.1`                                              |
| **Serbien-Seite**     | `10.0.0.2`                                              |
| **Details**           | → [`2.3-Firewall-OPNsense`](./2.3-Firewall-OPNsense.md) |
