## Topologie

<p align="center">
  <img
    src="../../assets/topology_rs.png"

</p>

Der Standort Serbien ist der zentrale Knotenpunkt meiner Infrastruktur. Hier läuft der Proxmox Core Node, die lokale Firewall, der PoE Switch sowie die WLAN-Versorgung. Alles wurde selbst aufgebaut und konfiguriert — von der Verkabelung bis zur Firewall-Regel.

<hr style="border: none; height: 2px; background: linear-gradient(to right, transparent, #555, transparent); margin: 30px 0;">

## Logischer Datenfluss

```text
[ Internet / ISP ]
        |
    [ Modem ]               ← ISP-Einspeisung
        |
   [ OPNsense ]             ← Firewall · NAT · VPN · Gateway (10.20.10.1)
        |
  [ PoE Switch ]            ← Zentraler Verteiler
   /    |    \
[Node] [AP1] [AP2]          ← Proxmox Core Node · Indoor AP · Outdoor AP
```

<hr style="border: none; height: 2px; background: linear-gradient(to right, transparent, #555, transparent); margin: 30px 0;">

## Netzwerk-Segmente

| Segment           | Netz            | Zweck                                      |
| ----------------- | --------------- | ------------------------------------------ |
| LAN Serbien       | `10.20.10.0/24` | Lokales Netzwerk — alle Geräte am Standort |

## VPN-Verbindung zu Deutschland

Der Standort Serbien ist über einen WireGuard Site-to-Site Tunnel mit dem Standort Deutschland verbunden. Die OPNsense in Serbien fungiert dabei als VPN-Endpunkt auf beiden Seiten.

| Parameter             | Wert                                                    |
| --------------------- | ------------------------------------------------------- |
| **Tunnel-Netz**       | `10.0.0.0/24`                                           |
| **Serbien-Seite**     | `10.0.0.2`                                              |
| **Deutschland-Seite** | `10.0.0.1`                                              |
| **Details**           | → [`2.3-Firewall-OPNsense`](./2.3-Firewall-OPNsense.md) |
